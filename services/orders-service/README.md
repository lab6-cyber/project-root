# Orders Service
Микросервис управления заказами и транзакциями.

Сервис отвечает за создание заказов, хранение истории покупок и взаимодействие с сервисом товаров для актуализации остатков на складе.

## Эндпоинты

### Заказы
- `POST /api/orders` — Создание нового заказа. Автоматически списывает количество купленного товара через Products Service.
- `GET /api/orders?userId=ID` — Получение списка заказов конкретного пользователя.
- `GET /api/orders?all=true` — Получение всех заказов в системе (доступно для роли Admin).
- `PATCH /api/orders/:id/status` — Изменение статуса заказа (например: "В пути", "Доставлен"). Используется администратором.

## Стек технологий
- Node.js
- Express
- PostgreSQL
- Axios (для межсервисного взаимодействия)

## Инструкция по запуску (Локально)

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Настройте переменные окружения в файле `.env` (используйте `.env.example` как шаблон).

3. Запустите сервис:
   ```bash
   npm start
   ```

4. Запуск тестов:
   ```bash
   npm test
   ```
